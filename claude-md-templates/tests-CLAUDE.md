# tests/

All test files. Tests use TDD (Red-Green-Commit).

## Conventions

- File naming: `test_<system>.gd` (e.g., `test_vehicle_spawn.gd`)
- Mirror magic numbers from production code as named constants
- Use test helper classes from `tests/helpers/`
- Both unit AND integration tests required for most changes
