# Changelog Generation

## When to Use

- **After release milestones** — generate a changelog for a tagged release
- **Before PR descriptions** — summarize what changed since the last release
- **For community posts** — distill changes into player-friendly language for Discord, social media, or store updates

## How It Works

This project uses **conventional commits** (`type: description`). Changelogs are generated by parsing `git log`, grouping entries by type, and formatting them into the appropriate template.

### Commit Type Mapping

| Commit Type | Changelog Section | Player-Facing? |
|-------------|-------------------|----------------|
| `feat` | Added | Yes |
| `fix` | Fixed | Yes |
| `perf` | Performance | Yes |
| `refactor` | Changed | Sometimes |
| `docs` | Documentation | No |
| `test` | Testing | No |
| `ci` | CI/CD | No |
| `chore` | Maintenance | No |
| `build` | Build | No |
| `style` | Style | No |

## Command Recipes

### Raw log since last tag

```bash
git log $(git describe --tags --abbrev=0 2>/dev/null || git rev-list --max-parents=0 HEAD)..HEAD --oneline --no-merges
```

### Grouped by type (feat/fix/etc)

```bash
git log $(git describe --tags --abbrev=0 2>/dev/null || git rev-list --max-parents=0 HEAD)..HEAD \
  --oneline --no-merges \
  --pretty=format:"%s" \
  | sort -t: -k1,1
```

### Between two tags

```bash
git log v0.2.0..v0.3.0 --oneline --no-merges --pretty=format:"- %s"
```

### Count commits by type

```bash
git log v0.2.0..HEAD --oneline --no-merges --pretty=format:"%s" \
  | sed 's/:.*//' | sort | uniq -c | sort -rn
```

## Templates

### Keep a Changelog (CHANGELOG.md)

Follows [keepachangelog.com](https://keepachangelog.com/) format.

```markdown
# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [0.3.0] - 2026-XX-XX

### Added
- Race countdown sequence with 3-2-1-GO visual and audio cues
- Lap timing system with checkpoint validation
- Live HUD showing current lap, best lap, and lap counter

### Fixed
- Wheel terrain detection no longer snags on mesh seams
- Ghost replay no longer desyncs after checkpoint reset

### Changed
- Suspension math extracted to pure static class for testability

### Performance
- Object pooling for dust particle effects (eliminated GC spikes)

[0.3.0]: https://github.com/Totes-MickGOATs/r8eo-x-unity/compare/v0.2.0...v0.3.0
[Unreleased]: https://github.com/Totes-MickGOATs/r8eo-x-unity/compare/v0.3.0...HEAD
```

## Workflow Integration

1. **During development:** Conventional commits accumulate automatically
2. **At release time:** Run the git log recipes above to collect changes
3. **Generate changelog:** Fill in the Keep a Changelog template
4. **Write announcements:** Adapt the Discord and store templates — keep only player-facing changes
5. **Tag the release:** `git tag -a v0.X.0 -m "Release v0.X.0 — Milestone Name"`
6. **Publish:** Push tag, create GitHub Release, post announcements
