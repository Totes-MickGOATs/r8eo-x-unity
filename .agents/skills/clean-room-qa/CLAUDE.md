# clean-room-qa/

Independent black-box QA testing skill. Writes tests with ZERO implementation knowledge — only function signatures and domain physics.

## When to Use
- Porting code between engines (Godot → Unity)
- Verifying physics/math behaves correctly by first principles
- Auditing systems you didn't write
- `/clean-room-qa <system-name>` to invoke

## Key Principle
> If you know how the code works, you can't objectively test whether it works correctly.
